# sonarfix-ai

**Automated vulnerability remediation suggester for SonarQube reports.**

Parses SonarQube JSON output, classifies findings by severity, and queries a Large Language Model to generate concrete, developer-friendly remediation suggestions for each issue — reducing the manual effort required to triage audit results.

---

## Overview

Code audit tools like SonarQube surface a large volume of security findings, but development teams rarely have the bandwidth to investigate each alert and determine exactly what needs to change. This tool bridges that gap by automating the triage and explanation step: given a SonarQube report, it produces a prioritised list of findings with actionable fix instructions generated by an LLM.

The pipeline is straightforward:

```
SonarQube JSON report
    → Parse and classify by severity (CRITICAL / MAJOR / MINOR)
    → Build context-aware prompt per vulnerability
    → Query LLM API
    → Print structured remediation output
```

```
[ 1/5 ]  CRITICAL  —  SQLI-INJECTION
  File    : src/api/user_controller.py  (line 42)
  Finding : User input is directly concatenated into SQL query without sanitization.

  Generating remediation suggestion...

  This vulnerability allows an attacker to manipulate the SQL query by injecting
  arbitrary input, potentially exposing or deleting the entire database contents.
  In a production environment, this could lead to full data breach or service outage.

  Fix: Replace string concatenation with a parameterised query:
      query = 'SELECT * FROM users WHERE id = %s'
      cursor.execute(query, (user_id,))

  Priority: Fix immediately — SQL injection is a critical, well-understood attack
  vector that is trivial to exploit once discovered.
```

## Usage

```bash
# Clone the repository
git clone https://github.com/Asmasaaif/sonarfix-ai.git
cd sonarfix-ai

# Set your Anthropic API key
export ANTHROPIC_API_KEY="your-api-key-here"   # Linux / macOS
set ANTHROPIC_API_KEY=your-api-key-here         # Windows

# Run against the sample report
python sonarfix.py sample_report.json

# Run against your own SonarQube export
python sonarfix.py your_report.json
```

## Design Notes

The prompt sent to the LLM is deliberately open-ended rather than template-driven. Asking the model to explain the risk, provide a fix, and assess urgency in natural language produces output that is more readable for developers than a rigidly formatted response. The severity classification logic mirrors standard vulnerability management triage: CRITICAL findings are surfaced first to ensure the highest-risk issues are addressed before lower-severity items.

In a production context, this pipeline would be extended to include retry logic for rate-limited API responses, caching to avoid redundant calls for identical rule types, and structured JSON output for integration with ticketing or CI/CD systems.
